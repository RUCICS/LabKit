# 本地认证与 Evaluator 示例设计

## 目标

补一套中文参考资料和可运行示例，让开发者在学校 OAuth 尚未接通时，仍然能在本地把以下链路跑通：

- CLI 发起设备授权
- 使用 dev bind 完成本地认证
- 注册一个测试 Lab
- 构建并配置 evaluator 镜像
- 提交文件并在排行榜里看到结果

## 设计范围

这次不修改核心鉴权协议，不新增新的后端接口，也不重做 worker runtime。只做三类交付物：

1. 中文参考文档
   说明当前学生登录、浏览器 session、CLI 签名鉴权、admin token 鉴权、本地 dev auth workaround、manifest 约定、evaluator 镜像约定。

2. 可直接使用的示例文件
   提供一个测试用 manifest 和一个最小 evaluator 示例目录，减少“看懂了文档但还得自己拼第一版”的摩擦。

3. README 入口
   在仓库首页把这些文档和示例挂出来，避免信息散落在聊天记录里。

## 关键原则

- 文档必须中文，面向第一次接触仓库的人。
- 示例必须最小，但要真实可跑。
- 文档要明确区分“当前已经实现的行为”和“未来可能优化的体验”，避免把 workaround 写成正式 UX。
- evaluator 约定要写清楚到可以独立实现镜像，不依赖读源码猜接口。
